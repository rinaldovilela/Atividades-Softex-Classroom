Conhecendo o HTML e sua estrutura básica

1. O que é o HTML e qual é o seu papel na construção de páginas web? Descreva brevemente a estrutura básica de um documento HTML.

R:HTML significa Hypertext Markup Language ou linguagem de marcação. É a linguagem padrão para criação de páginas web. Seu papel é definir a estrutura e o conteúdo das páginas.

2. Como o HTML se relaciona com outras tecnologias da web, como o CSS e o JavaScript? Como essas três tecnologias trabalham em conjunto para criar páginas web interativas e estilizadas?

R:O HTML, o CSS e o JavaScript são as três tecnologias fundamentais da web. O HTML é responsável pela estrutura e o conteúdo, o CSS cuida da apresentação e do estilo, e o JavaScript adiciona interatividade às páginas. Juntas, essas tecnologias trabalham em conjunto para criar páginas web interativas e estilizadas.

Identificando tags HTML básicas

1. Cite três tags HTML básicas e explique o propósito de cada uma delas. Dê exemplos de situações em que você as usaria ao construir uma página web.

R: Três tags HTML básicas :

As tags : <h1>, <h3>, <h3> Usadas para criar cabeçalhos de diferentes níveis de importância.

A tag <p>: Usada para criar parágrafos de texto.

A tag <a>: Usada para criar links para outras páginas ou recursos.

2. Além das tags mencionadas, qual é a importância das tags de listagem, como <ul> e <ol>, no contexto da estruturação de conteúdo em uma página? Dê exemplos de quando você escolheria uma sobre a outra.

R:As tags de listagem, como <ul> (unordered list) e <ol> (ordered list), são importantes para organizar e estruturar conteúdo em uma página. <ul> cria uma lista não ordenada e <ol> cria uma lista ordenada. A escolha entre elas depende da ordem desejada dos itens. Pode se usar <ul> quando a ordem não importar e <ol> quando a ordem for significativa, como em listas numeradas.

Vinculando páginas, arquivos e âncoras

1. Como a tag <a> é utilizada para criar links em HTML? Explique como você pode vincular para outras páginas, para arquivos locais e criar âncoras dentro de uma página.

R:<a> define a criação de um link, href especifica o destino do link, que pode ser uma URL, um arquivo local ou uma âncora e dentro da tag é colocado o Texto do link.

2. Além do atributo href, quais outros atributos importantes a tag <a> pode ter? Como o atributo target pode ser usado para controlar como o link é aberto pelo navegador?

R:Alguns atributos que a tag <a> pode ter são : target, title, download, rel, herflang, type, class, id, style. O atributo target pode ter várias formas para controlar como o link é aberto pelo navegador e entre eles estão :

  • target="_blank": Abre o link em uma nova janela ou guia do navegador, mantendo a página atual aberta.

  • target="_self": Abre o link na mesma janela ou guia em que o link foi clicado (comportamento padrão), substituindo o conteúdo da página atual.

  • target="_parent": Abre o link na janela pai, útil quando se trabalha com estruturas de frames (framesets), substituindo o conteúdo da janela pai se houver frames.

  • target="_top": Abre o link na janela superior, substituindo todas as frames, se houver, permitindo sair completamente de estruturas de frames.

Incorporando imagens e mídias

1. Descreva a diferença entre os atributos src e alt em relação à tag <img>. Por que é importante fornecer um atributo alt adequado para imagens?

R: Por que quando a imagem não é carregada corretamente, aparecerá o texto colocado no atributo alt para ue o usuário identifique do que se trata aquele conteúdo .

2. Além de imagens, quais outros tipos de mídia podem ser incorporados em uma página HTML? Como você pode garantir que a mídia incorporada seja responsiva e se ajuste bem em diferentes dispositivos?

R:Além de imagens, outros tipos de mídia que podem ser incorporados em uma página HTML incluem áudio (como arquivos de áudio MP3 ou WAV) e vídeo (como arquivos de vídeo MP4 ou WebM). Além disso, também é possível incorporar elementos como gráficos vetoriais (SVG) e elementos interativos, como animações em formato GIF ou SVG, além de fontes personalizadas (tipografia) usando fontes da web (Web Fonts).

Para garantir que a mídia incorporada seja responsiva e se ajuste bem em diferentes dispositivos : Posso usar CSS com propriedades como "max-width: 100%" para imagens e "width: 100%" para vídeos e configurar "media queries" para adaptar o layout em diferentes tamanhos de tela por exemplo.

Criando tabelas básicas

1. Quais são as principais tags usadas para criar tabelas em HTML? Explique a diferença entre as tags <th> e <td>, e quando você as usaria.

R: As principais tags usadas para criar tabelas em HTML são: <table>: Define a tabela. <tr>: Define uma linha na tabela. <th>: Define um cabeçalho de coluna (table header). <td>: Define uma célula de dados (table data).

<th> É para cabeçalhos, títulos das colunas. <td> É para dados na tabela.

2. Embora tabelas sejam usadas para exibir dados tabulares, às vezes é recomendado evitá-las em favor de outras formas de apresentação de dados. Por quê? E quais são os cenários em que as tabelas são apropriadas?

R:Tabelas são recomendadas quando se precisa exibir dados tabulares, como informações numéricas ou textuais organizadas em linhas e colunas. No entanto, em cenários de design web e acessibilidade, tabelas podem ser evitadas em favor de outras formas de apresentação, como gráficos ou listas, para melhorar a usabilidade e a acessibilidade do conteúdo.

Criando formulários

1. Como a tag <form> é utilizada para criar formulários em HTML? Cite pelo menos três tipos diferentes de campos de entrada (<input>) que podem ser usados em um formulário.

R: A tag <form> é usada para criar formulários em HTML. Alguns tipos de campos de entrada ( <input>) incluem:

<input type="text"> para entrada de texto.

<input type="checkbox"> para caixas de seleção múltipla escolha.

<input type="radio"> para botões de caixa de seleção.

<input type="Password"> para para senhas.

2. Além dos campos de entrada, quais outros elementos podem ser incluídos em um formulário para melhorar a usabilidade e a experiência do usuário?

R: Além dos campos de entrada outros elementos podem ser incluídos em um formulário. Como :

<select> Usado para Lista suspensa

Rótulos <label> é usada para associar um rótulo a um elemento de formulário. O atributo for do <label> deve corresponder ao atributo id do elemento de formulário.

Agrupamento de elementos de formulário:

<legend> fornece uma legenda descritiva para o grupo.

<fieldset> define o bloco de agrupamento.

Técnicas avançadas de estruturação com elementos HTML5 semânticos

1. O que são elementos HTML5 semânticos e por que são importantes na estruturação de uma página? Cite três exemplos de elementos semânticos e explique quando você os usaria.

R:Elementos semânticos do HTML5 são aqueles que descrevem claramente seu significado de uma maneira que seja legível tanto para humanos quanto para máquinas.

Aqui estão três exemplos de elementos semânticos e quando eu os usaria :

  • <article> Utilizaria para representar uma composição autônoma e independente dentro de uma página. Para notícias por exeplo.
  • <section> Utilizaria para agrupar e estruturar semanticamente o conteúdo relacionado.
  • <aside> Utilizaria para informações adicionais, como barras laterais, anúncios, conteúdo relacionado, entre outros.

2. Além da semântica, como os elementos HTML5 podem afetar a acessibilidade das páginas web? Por que é fundamental considerar a acessibilidade ao escolher elementos de estruturação?

R: A Semântica HTML é importante para a indexação por mecanismos de busca, pois ajudam os robôs dos mecanismos de busca a entender melhor o conteúdo da página.As tags semânticas permitem que os mecanismos de busca identifiquem facilmente o título da página, o conteúdo principal e as seções secundárias. Isso ajuda a melhorar a relevância da página nos resultados de busca.

É fundamental considerar a acessibilidade ao escolher elementos de estruturação pois tornar sites, aplicativos, textos, plataformas, redes sociais, blogs e outras ferramentas mais acessíveis rompe barreiras e democratiza o acesso aos ambientes digitais, garantindo que todos os usuários tenham acesso às mesmas informações.
